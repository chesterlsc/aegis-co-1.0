# Aegis & Co. 1.0

First iteration of the Aegis & Co. landing page.

Aegis & Co. is positioned as the systems and operations company. The site is designed to communicate structure, clarity, control, and operational visibility. Unlike NAMI, which focuses on creative output and how businesses are seen, Aegis focuses on how businesses run behind the scenes.

## Direction

This version is built around a light base with darker system-led sections.

The goal of the design is to make Aegis feel:
- Precise
- Technical
- Clear
- Modern
- Operational
- Premium
- Easy to understand for clients

The page uses a hybrid visual system:
- Light sections for readability and scanning
- Dark sections for system visuals, diagrams, and proof-based presentation

## Key Design Principles

- One clear job per section
- Calm and structured visual hierarchy
- Technical without being overwhelming
- System-oriented, but still client-friendly
- Minimal, modern, and business-ready
- Darker palette used only where visuals need stronger emphasis

## Current Structure

The site currently follows this layout:

1. Hero  
2. What Aegis Builds  
3. How It Connects  
4. Outcomes  
5. How the Work Moves  
6. System Areas We Work In  
7. Closing CTA  
8. Footer  

## Sections Overview

### Hero
Introduces Aegis as the operational layer that helps businesses run with less drag and more control.

Includes:
- Main headline
- Support copy
- Primary and secondary CTAs
- System status detail
- Support feature row
- Orchestration-style flowchart visual

### What Aegis Builds
Shows the main service areas in a modular card layout:
- Workflows
- Automations
- Reporting
- Internal Tools

### How It Connects
The main proof section of the page.

Shows how systems move through a logic path from input to output using a dark visual panel. This section is intended to make Aegis feel practical and systems-led rather than abstract.

### Outcomes
Communicates the business improvements Aegis brings:
- Less manual work
- Cleaner handoffs
- Faster reporting
- Better visibility
- Stronger control
- Fewer bottlenecks

### How the Work Moves
Shows the Aegis process in a simple step-based format:
- Map
- Design
- Build
- Test
- Improve

### System Areas We Work In
Shows where Aegis operates inside the business:
- Lead Routing
- Approvals
- Dashboards
- Reporting
- Ops Workflows
- Internal Tooling

### Closing CTA
Ends the page with a direct systems-focused close built around clarity, control, and results.

### Footer
Includes:
- Brand block
- Navigation
- Capabilities
- Contact
- System status detail

## Visual System

### Light Mode Base
Used for:
- Readability
- Section scanning
- Card sections
- Process sections
- Capability grids

### Dark Mode Accents
Used for:
- Hero system visuals
- Proof and process map sections
- Technical diagrams
- Stronger contrast moments

### Accent Colors
The palette is built around:
- Command Blue
- Process Green
- Neutral cool grays
- Restrained warning tones where needed

## Design Details Carried Into This Version

This version intentionally includes smaller system-style details that support the brand direction, such as:
- All Systems Operational
- SYS_01 style labels
- Status markers
- Icon-supported trust lines
- System status in the footer
- Structured flowchart visuals

## Core Message

Aegis is about freeing teams to do higher leverage work.

The design direction supports the idea that businesses should spend less time on manual work and more time on decisions, control, and forward movement.
